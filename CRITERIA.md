## Критерий 1 — Актуальность / инновационность + обоснованность применения AI

**Актуальность (якорь).** Утечка **16,3 млн ИИН** (>75% взрослого населения РК, июнь 2025).
По заключению **МЦРИАП и КНБ — не взлом периметра, а авторизованный доступ**: инсайдер либо
скомпрометированные креды. Запрос к базе выглядит легально → **DLP / SIEM / файрвол это не видят**.
Контекст IBM 2025: malicious insider — самый дорогой вектор ($4,92M), средний MTTD — **241 день**.
Это не абстрактная угроза, а конкретная боль гос-органов РК, под которую построен продукт.

**Инновационность (новизна vs существующие классы).**

- **DLP/SIEM** ловят сигнатуры, периметр, известные паттерны — у инсайдера доступ **легальный**,
  сигнатуры нет.
- **Insider Guard** строит детект на **отклонении от персональной нормы самого пользователя**
  (UEBA): объём, время, широта доступа, привилегии, гео/устройство. Norma считается **из истории
  пользователя методом leave-one-day-out** (аномальный день не раздувает свой же baseline) и
  относительно медианы коллег по роли (perRole). Это `server/engine.js` — computed baseline,
  **не хардкод**.
- **Объяснимость как ядро, а не пост-фактум.** Аддитивная декомпозиция «почему помечено»
  (Hamilton / largest-remainder, `server/lib/shap.js`), где **Σ вкладов триггеров === score
  математически точно**, бит-в-бит, проверяемо (`docs/contract.md`, бейдж «Σ = score (exact)»
  в UI). Это не вызов библиотеки `shap`, а точная целочисленная декомпозиция весов — семантика
  та же (вклад фактора в решение), но воспроизводимая и аудируемая.

**Обоснованность AI.** AI применён ровно там, где он нужен и где детерминированные правила
бессильны: распознать «×N от собственной нормы» на потоке, где разметки почти нет
(imbalance ~10⁻⁶). Там же, где AI опасен (источник вердикта), он **исключён архитектурно** —
LLM строго поверх посчитанного результата (см. критерий 2). Это осознанный, защищаемый выбор.

**Доказательства:** `server/engine.js` (baseline + триггеры), `server/lib/shap.js` (Σ===score),
`GET /api/incidents/:id` (граф + SHAP + baseline + playbook в одном ответе), README §«Якорь 16,3 млн».

## Критерий 2 — Наличие AI-модели, НЕЗАВИСИМОЙ от иных сервисов

> Это критерий, на котором проще всего потерять баллы из-за «AI = вызов чужого API».
> У нас детект делает **собственная модель на устройстве**, а внешний LLM — опциональная косметика.

**Собственная unsupervised anomaly-модель — на устройстве, без внешних сервисов.**

- **Robust modified z-score (median / MAD).** Реализовано в `server/lib/fmt.js` →
  `madZScores()` и применяется в `server/engine.js` → `peerGroupMad()`. Это **unsupervised
  anomaly detection без меток**: ранжируем пользователей внутри peer-group (по роли) по дневному
  объёму чтений через **модифицированный z-счёт на медиане и MAD** (`0.6745/MAD`), с корректной
  деградацией на MeanAD при вырожденном MAD. Сигнал «×N от своей **персональной** нормы И ×M от
  **медианы peer-group**» — это статистическая anomaly-модель, а не if-then-список.
- **Почему MAD, а не нейросеть.** Осознанный выбор под imbalance ~10⁻⁶, где DL/GNN
  переобучается на десятках примеров. Робастный MAD-z устойчив к выбросам (медиана/MAD, не
  mean/σ), работает **real-time без GPU**, **детерминированно** (тот же вход → тот же score
  бит-в-бит), а значит **аудируемо** и с нулём галлюцинаций. Для регуляторного домена
  (АФМ, ПДн) детерминизм — преимущество, а не компромисс.
- **Полная независимость от внешних сервисов.** В `package.json` **нет** ML-зависимостей,
  нет sklearn/torch, **нет вызовов внешних API в пути детекта**. Движок — pure-функции
  (`server/engine.js`, без DB, без сети). Отключи интернет — детект, score, SHAP, граф,
  очередь, метрики работают полностью. Запускается on-prem за периметром.
- **Детерминированный seeded-корпус.** Демо-данные генерятся seeded-PRNG (`npm run seed`) →
  воспроизводимы; конфиг модели (веса, VOLUME_MULT, окно baseline) снапшотится в `run_meta`
  на каждый датасет — модель воспроизводима и аудируема.
- **Точка интеграции Isolation Forest — за флагом, drop-in.** Robust-MAD относится к тому же
  семейству unsupervised anomaly-detection, что и Isolation Forest; IF заготовлен как
  опциональная замена за флагом и **не требуется** для работы детекта. Заявляем ровно это —
  чтобы на Q&A фраза была неуязвима.

**Внешний Claude — ТОЛЬКО опциональный черновик отчёта, с offline-fallback.**

- `claude-opus-4-8` используется **исключительно** для investigation-нарратива / черновика
  IR-отчёта (`server/report.js`, `POST /api/report/:id`) — **никогда не считает score / вердикт**
  (anti-pattern «LLM as source of truth» исключён архитектурно).
- По умолчанию черновик генерится **детерминированным mock-шаблоном** (`mockReport`, работает
  оффлайн); `claudeReport` обёрнут в try/catch → mock-fallback. **Демо не зависит от сети/ключа.**
- Итог формулировки для жюри: **«Детект делает наша собственная unsupervised-модель на
  устройстве, без единого внешнего сервиса. Claude — опционально и только для текста отчёта,
  и даже его можно выключить.»**

**Доказательства:** `server/lib/fmt.js:78` (`madZScores`), `server/engine.js` (`peerGroupMad`,
строка ~963), `package.json` (нет ML/AI-зависимостей в пути детекта), `server/report.js`
(LLM только в отчёте + mock-fallback).

## Критерий 3 — Информационная безопасность + соблюдение этических норм

**Информационная безопасность (см. `SECURITY.md`).**

- **Threat-model OWASP Top-10 + STRIDE** пройден; найденные риски устранены.
- **Фикс multer / upload-вектора:** валидация типа и размера загружаемого лога, защита от
  необработанного payload на `POST /api/ingest`.
- **Строгая валидация ingest-контракта:** обязательные поля (`user`, `action`, `ts`),
  безопасный парсинг `ts` (ISO / `YYYY-MM-DD HH:MM` / дата / epoch), дефолты вместо падения.
- **Атомарная персистентность** (temp-файл + `rename`) — краш в середине записи не портит стор;
  **graceful degradation** на read-only FS (не падаем).

**Этические нормы и privacy-by-design.**

- **Синтетика, 0 реальных ИИН** — на экране только сгенерированные данные; в коде/корпусе нет
  реальных персональных данных.
- **Privacy-by-design:** псевдонимизация пользователей, per-user baseline (anti-bias — мерим
  человека относительно него самого, а не относительно «подозрительной группы»), данные не
  покидают периметр (on-prem).
- **Human-in-the-loop (EU AI Act ст. 14):** система **приоритизирует и объясняет**, но **решает
  человек**. Никогда не утверждаем «доказанный инсайдер» — только «аномалия + объяснение для
  проверки аналитиком». Tiered-autonomy actions (AUTO / APPROVE / HIGH-RISK) — рискованные
  действия требуют подтверждения человека.
- **Tamper-seal / chain-of-custody:** SHA-256-печать досье (правка одного числа → `INTEGRITY
  BROKEN`) + append-only hash-chain audit-log → действие воспроизводимо бит-в-бит и пригодно
  как доказательство для следствия, а аналитик не может «тихо» подменить данные.

**Доказательства:** `SECURITY.md` (OWASP/STRIDE, multer-фикс), валидация в `server/engine.js` /
ingest-роуте, SHA-256 tamper-seal в UI + экспорт `.json`, hash-chain audit-log.

## Критерий 4 — Практическая реализуемость + масштабируемость

**Реализуемость — это рабочий MVP, а не слайды.**

- **Живой деплой:** https://insider-guard.vercel.app — детект, очередь, граф, SHAP, метрики,
  tamper-seal работают прямо сейчас.
- **Гарантированный локальный deliverable:** `npm install && npm run seed && npm start` →
  http://localhost:3000. Реальный движок + seeded-корпус 40 пользователей × 30 дней.
- **Контрактные тесты:** `npm run test:contract` доказывают форму каждого эндпоинта.

**Масштабируемость / интеграция с реальными источниками РК.**

- **Канонический 12-полевой event-контракт** (`user, role, resource, db, host, ip, geo, action,
  rows, ts, channel, label?`) + слой алиасов заголовков (`timestamp/time`, `table/target`,
  `username/account`, `src_ip`, `location`, …). Новый источник = **~50 строк адаптера**
  (AD / Colvir / ЦФТ / СУБД-аудит / 1С) — маппинг полей, не переписывание движка.
- **Загрузка своего CSV access-лога → baseline из ваших данных** без предразметки: судья
  перетаскивает свой лог во фронт или шлёт `POST /api/ingest`, движок считает per-user baseline
  (leave-one-day-out) из *этих* событий и поднимает аномалию в топ. Сэмпл:
  `samples/insider-access-log.csv` (синтетика).
- **On-prem / суверенитет:** Закон №94-V о локализации ПДн РК (в силе с 08.01.2025). On-prem —
  не «фича», а причина, по которой SaaS-UEBA гос-заказчику структурно непродаваем. Детект без
  внешних сервисов (критерий 2) делает on-prem реальным.
- **Архитектура под рост:** движок — stateless pure-функции, скорит массив событий любого
  размера; персистентность через volume; конфиг снапшотится на датасет.

**Доказательства:** live demo, `npm run seed`/`npm start`, `docs/contract.md` (12-полевой
контракт + алиасы), `POST /api/ingest`, `samples/insider-access-log.csv`.

## Критерий 5 — Качество презентации и технической защиты (MVP)

- **Демо укладывается в окно:** прогон **2:47** при лимите **защита 3 мин + Q&A 2 мин** —
  есть запас на вопросы.
- **`DEMO-SCRIPT.md`** — поминутный сценарий с таймингами, репетицией и контингенциями
  (что резать первым при цейтноте).
- **`JUDGE-CHEATSHEET.md`** — готовые ответы на острые вопросы жюри (где AI, почему не accuracy,
  почему не нейросеть, независимость модели, ИБ/этика).
- **Отполированный UI:** премиальный тёмный дашборд, kill-chain Play-скраббер, MITRE-бейджи,
  SHAP-водопад с бейджем «Σ = score (exact)», tamper-seal с визуальным `INTEGRITY BROKEN`.
- **Доказательство точности на защите:** числа фиксируются вживую из `GET /api/metrics`
  (порог-слайдер пересчитывает confusion-matrix). Формулировка — **«N инцидентов / типологии
  покрыты / benign-контроли не помечены»**, метрики **recall@top-N / precision / AUPRC**,
  явно **НЕ accuracy** (пустышка «всё чисто» на дисбалансе даёт ~99,9% accuracy и ловит ноль).

**Доказательства:** `DEMO-SCRIPT.md`, `JUDGE-CHEATSHEET.md`, live UI, `GET /api/metrics`.
